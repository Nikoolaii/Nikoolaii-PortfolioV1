Installation Galera

Pour créer une continuité avec Galera, il faut créer 3 VM qui auront ces spécificité :

  • 20Gb
  • 1Gb ram

Les serveurs auront 3 ip différents :

  • ServBDD1 : 192.168.88.152
  • ServBDD2 : 192.168.88.152
  • ServBDD3 : 192.168.88.152

Le cluster de BDD auront comme mot de passe:

Mot de passe root : aaa

Mot de passe aaa (compte non admin) : aaa

Installation de MariaDB sur les machines

Pour installer MariaDB, il faut executer cette commande dans le terminal : 

sudo apt-get install mariadb-server

Il faut par la suite, le lancer et activer le lancement au démarrage 

sudo systemctl start mariadb
sudo systemctl enable mariadb
sudo systemctl status mariadb

Normalement galera c’est installé de base sur les machines, si c’est pas le cas il faut executer cette commande : 

sudo apt-get install mariadb-server-galera

Mise en place sur le serveur 1

Sur votre premier serveur, donc votre serveur principal, vous devrez executer la commande

sudo nano /etc/my.cnf.d/galera.cnf

Vous devrez modifier

Vous devrez ensuite démarrer galera sur la VM 1 uniquement

galera_new_cluster

Vous devrez aussi ajouter les IP des autres serveurs dans le fichier conf de galera

Sur les autres serveur, vous allez devoir indiquer les même paramètres que sur le premier :

Node 2 :

# Logical cluster name. Should be the same for all nodes.
wsrep_cluster_name="cluster_alger"

# Group communication system handle
wsrep_cluster_address="gcomm://192.168.88.152,192.168.88.153,192.168.88.154”

# Human-readable node name (non-unique). Hostname by default.
#wsrep_node_name=

# Base replication <address|hostname>[:port] of the node.
# The values supplied will be used as defaults for state transfer receiv>
# listening ports and so on. Default: address of the first network inter>
wsrep_node_address=192.168.88.153

Node 3 :

# Logical cluster name. Should be the same for all nodes.
wsrep_cluster_name="cluster_alger"

# Group communication system handle
wsrep_cluster_address="gcomm://192.168.88.152,192.168.88.153,192.168.88.154”

# Human-readable node name (non-unique). Hostname by default.
#wsrep_node_name=

# Base replication <address|hostname>[:port] of the node.
# The values supplied will be used as defaults for state transfer receiving,
# listening ports and so on. Default: address of the first network interface.
wsrep_node_address=192.168.88.154

Il faut ensuite redémarrer les services MariaDB

sudo systemctl enable --now mariadb

Pour vérifier la synchronisation:

show status like 'wsrep_%'

Pour finir il faut ensuite ouvrir les différents nécéssaire à l’utilisation de la BDD

firewall-cmd --add-port=4567/tcp --permanent
firewall-cmd --add-port=3306/tcp --permanent
firewall-cmd --add-port=4444/tcp --permanent
firewall-cmd --add-port=4568/tcp --permanent
firewall-cmd --add-port=80/tcp --permanent

firewall-cmd --reload
firewall-cmd --list-all
