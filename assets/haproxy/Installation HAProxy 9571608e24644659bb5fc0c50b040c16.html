    Mise en place des machines virtuelles pour les serveurs.

La première étape est d’utiliser un logiciel qui permet la création de machines virtuelles, ici ce sera VMWare

Ce qui est important dans la création de cette VM, c’est de mettre 2 accès réseau au serveur web et au serveur HAProxy.

Voici donc les paramètres techniques des VM.

Pour le bien ce ce TP, il est possible de créer uniquement 1 VM qui sera cloné en 2 autres VM.

Au lancement de la machine virtuelle, il faut choisir « Install ».

Après quelques choix comme celui de la langue ou de la région, vous devrez entrez les login choisis pour la connexion à votre VM, vous aurez cette page qui s’affiche, vous devrez choisir ces options :

Une fois l’installation terminée, vous aurez cette page qui s’affiche, vous pouvez ensuite éteindre la VM et la clonée

Mise en place des serveurs apache.

Avant de procéder à l’installation des autres applications, on doit d’abord configurer le fichier sources.list que l’application « apt » utilise pour chercher les paquets correspondants.

Le fichier sources.list, qui se trouve dans le répertoire /etc/apt/ contient les adresses des sites miroir qui hébergent ces paquets. Pour cela on utilisera l’éditeur de texte nano.

Il faut aussi attribuer une adresse ip au serveur

Pour cela, il vous faut entrez la commande « ip a »

Vous allez devoir récupérer les 2 nom d’interfaces et les configurer avec une ip en écrivant

« nano /etc/network/interfaces »

Et entrez ces valeurs sur la VM1 donc le serveur HAProxy :

Il faudra ensuite activer les interfaces en entrant les commandes « ifup ens33 » et « ifup ens36 »

Pour vérifier le bon fonctionnement, essayer avec un « ping free.fr ».

Vous pourrez alors ensuite cloner la machine virtuelle.

Ce clone deviendra donc VM2, qui sera le premier serveur web, sur celui-ci, il vous faudra exécuter un « apt install apache2 », en appuyant sur O pour valider, apache2 va donc s’installer.

Une fois l’installation faites, vous aurez juste à cloner VM2, ce qui vous donnera VM3, une fois cela fait, vous devrez changer leurs adresses ip, grâce à la commande « nano /etc/network/interfaces » en changeant l’adresse de l’interface ens33.

Il faudra ensuite relancer l’interface sur VM2 et VM3 en exécutant la commande « ifdown ens33 » et « ifup ens33 » sur les 2 machines.

Il faut aussi préciser que les adresses ip doivent être différentes sur les 3 machines.

Vous pourrez ensuite tester vos pages, pour cela en tapant la commande « ip a », vous trouverez cette ip, qui une fois entrée dans le navigateur vous affichera une page web :

Vous pouvez le faire sur les VM2 et VM3

Modifier les pages html pour différencier les serveurs.

Il est nécessaire de modifier les pages html pour pouvoir les différencier lors du test HAProxy.

Pour cela, vous allez devoir exécuter la commande « nano /var/www/html/index.html »

Ici vous pourrez modifier la page pour obtenir 1 affichage différent sur chacun de vos serveurs.

Mise en place du HAProxy

Sur la VM1, vous devrez exécuter les commandes « apt update », « apt upgrade », « apt install -y apache2 » et « apt install -y haproxy ».

Vous devrez ensuite ajouter l’adresse ip de vos serveurs web dans :

« nano /etc/hosts/ »

Et ensuite aller dans la configuration de votre serveur haproxy en tapant « nano /etc/haproxy/haproxy.cfg »

Vous écrivez par la suite dans le terminal du SRV1 donc le HAPRoxy « systemctl reload haproxy »
